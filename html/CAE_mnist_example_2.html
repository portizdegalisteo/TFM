Ejemplo de entrenamiento de CAE sobre MNIST 2¶

Este notebook tiene dos objetivos:

  • Entrenamiento a partir de imgenes PNG: uno de los problemas encontrados a la hora de entrenar una red con imgenes es el tamao del conjunto de entrenamiento. Una gran cantidad de imgenes de resolucin media pueden saturar la memoria RAM de la mquina y parar el programa. Para evitar esto Keras tiene mtodos implentados que hacen el entrenamiento por batches leyendo nicamente un subconjunto de imgenes en cada momento. El primer objetivo es analizar esta posibilidad.

  • Error de reconstruccin como prediccin: adems de para extraer variables latentes se le pueden dar otros usos al Autoencoder Convolucional. En este caso se pretende entrenar dos CAE con subconjuntos de dgitos distintos (eg. 0,1,2,3,4 vs 5,6,7,8,9), y sobre el conjunto de test calcular el error de reconstruccin de cada imagen en ambas redes. Conceptualmente un nmero del primer conjunto tendr un menor error de reconstruccin con el primer modelo que con el segundo.

Generador de Imágenes¶

Para realizar el entrenamiento por batches que se menciona al incicio de este notebook se utiliza la clase ImageDataGenerator, esta sirve para crear un lector de imgenes y evitar cargar todas a la vez en memoria. Esta clase tiene un mtodo flow_from_dataframe donde se crea el lector de imgenes a partir de los nombres de ficheros que haya en un DataFrame.

Tambin permite crear un lector de imgenes para el conjunto de validacin o, por ejemplo, redimensionar todas las imgenes al un tamao concreto target_size, esto sera muy util si las imagenes disponibles no tuviesen dimensiones constantes. En este caso mantenemos las originales 28x28.

IMPORTANTE: es necesario reescalar el rango de valores de imgenes entre 255 ya que por defecto cuando see leen las imgenes RGB los valores de los pxeles van de 0 a 255, el nivel en la escala de 8 bits. Con esos rangos de valores y los learning rate que hay por defecto el modelo sera incapaz de aprender.

Entrenamiento¶

El diseo de la arquitectura del CAE se utiliza un nmero bajo de variables latentes. La idea detrs de esto es que todas las imgenes de dgitos tienen propiedades muy similares (bordes, color...) que podran hacer que un CAE entrenado con unos dgitos sea vlido para otros. Al un nmero pequeo de variables latentes el error de reconstruccin ser mayor pero se est forzando al modelo a ser ms especfico. En este caso lo importante no es minimizar el error de reconstruccin si no maximizar la diferencia de error de un modelo respecto del otro.
